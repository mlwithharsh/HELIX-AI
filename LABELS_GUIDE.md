# GitHub Label System

A structured label system ensures that issues are triageable and accessible to all skill levels.

## Priority Labels
- **p0: Critical**: System-breaking issues that require immediate attention.
- **p1: High**: Major features or significant bugs impacting several users.
- **p2: Medium**: Standardization and general enhancements.
- **p3: Low**: Minor tweaks, refactoring, or documentation updates.

## Complexity Labels
- **beginner**: Simple tasks requiring minimal understanding of the core architecture.
- **intermediate**: Tasks involving logic updates across multiple modules.
- **advanced**: Structural changes, performance critical path, or new agent integration.

## Type Labels
- **bug**: Functional errors in the codebase.
- **enhancement**: New features or improvements to existing logic.
- **documentation**: Updates to markdown files or code comments.
- **maintenance**: Dependency updates, CI/CD improvements, or technical debt.

## Onboarding Labels
- **good-first-issue**: Reserved for beginner-friendly tasks that are clearly defined and limited in scope.

## How to use
- Every issue must have at least one **Priority**, one **Complexity**, and one **Type** label.
- Only project maintainers should assign **Priority** labels.
- Contributors can propose **Complexity** labels in their feature requests.
